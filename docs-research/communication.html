2. MAVLink Protocol

MAVLink là giao thức nhị phân nhẹ cho giao tiếp drone. QGC hỗ trợ cả MAVLink v1 và v2.

MAVLink Message Structure:

// MAVLink v2 Frame Format: ┌────┬────┬────┬────┬─────┬─────┬─────┬─────┬──────────┬────────┬──────────┐ │0xFD│Len │IFl │CFl │ Seq │SysID│CompID│MsgID│ Payload │Checksum│Signature │ │ 1B │ 1B │ 1B │ 1B │ 1B │ 1B │ 1B │ 3B │ 0-255B │ 2B │ 13B(opt)│ └────┴────┴────┴────┴─────┴─────┴─────┴─────┴──────────┴────────┴──────────┘ // 0xFD = MAVLink v2 start marker (0xFE = v1) // SysID = System ID (1-255, identifies vehicle) // CompID = Component ID (autopilot, camera, gimbal, ...) // MsgID = Message type (3 bytes = 16.7M possible messages) // Signature = Optional MAVLink2 signing for security

Message Processing Pipeline:

// 1. Bytes arrive from LinkInterface LinkInterface::bytesReceived(link, data) │ ▼ // 2. MAVLinkProtocol parses raw bytes MAVLinkProtocol::receiveBytes(link, data) ├── mavlink_parse_char() // C library function ├── Track sequence numbers // Detect packet loss ├── Check for new vehicles // HEARTBEAT = new vehicle? │ └── emit vehicleHeartbeatInfo() ├── Log to file // Save .mavlink log └── emit messageReceived(link, message) │ ▼ // 3. Vehicle processes message Vehicle::_mavlinkMessageReceived(link, message) ├── switch(message.msgid) │ ├── HEARTBEAT → _handleHeartbeat() │ ├── SYS_STATUS → _handleSysStatus() │ ├── ATTITUDE → _handleAttitude() │ ├── GPS_RAW_INT → gpsFactGroup.handleMessage() │ ├── GLOBAL_POSITION → _handleGlobalPosition() │ ├── PARAM_VALUE → parameterManager.handleMessage() │ ├── MISSION_* → missionManager.handleMessage() │ ├── STATUSTEXT → statusTextHandler.handleMessage() │ └── ... │ └── FirmwarePlugin specific handling

3. LinkInterface (Abstract Base)

Tất cả communication links kế thừa từ LinkInterface.

Interface chính:

class LinkInterface : public QThread { Q_OBJECT public: virtual bool _connect() = 0; // Establish connection virtual void disconnect() = 0; // Close connection virtual bool isConnected() = 0; // Connection status void writeBytesThreadSafe(const QByteArray&); // Thread-safe write uint8_t mavlinkChannel(); // Unique MAVLink channel (0-255) signals: void bytesReceived(LinkInterface*, QByteArray); void bytesSent(LinkInterface*, QByteArray); void connected(); void disconnected(); void communicationError(const QString& title, const QString& error); protected: virtual void _writeBytes(const QByteArray&) = 0; // Actual write };

Shared Pointer Pattern:

// Links sử dụng shared_ptr để quản lý lifetime an toàn using SharedLinkInterfacePtr = std::shared_ptr<LinkInterface>; // LinkManager::createConnectedLink() trả về shared_ptr // Vehicle giữ shared_ptr → link sống miễn vehicle còn reference // Khi tất cả references giải phóng → link tự cleanup

6. MAVLink Channel System

Mỗi link được cấp phát một MAVLink channel unique (0-255). Channel cô lập MAVLink state giữa các links.

// MAVLink C library duy trì state riêng cho mỗi channel: // - Sequence number (để detect packet loss) // - Parse state (multi-byte message assembly) // - Signing state (MAVLink v2 security) // Channel allocation: // LinkManager::allocateMavlinkChannel() → free channel từ pool // LinkManager::freeMavlinkChannel() → trả channel về pool // Multi-vehicle scenario: // Link 1 (Serial, channel 0) → Vehicle 1 (sysid 1) // Link 2 (UDP, channel 1) → Vehicle 2 (sysid 2) // Link 3 (UDP, channel 2) → Vehicle 1 (redundant link!)

7. VehicleLinkManager (Per-Vehicle)

Mỗi Vehicle có VehicleLinkManager riêng để quản lý 1+ links.

VehicleLinkManager │ ├── primaryLink // Link chính (ưu tiên gửi commands) ├── linkNames[] // Tên tất cả links ├── linkStatuses[] // Status mỗi link │ ├── communicationLost // true khi không nhận heartbeat > 3.5s ├── communicationLostEnabled // Có bật detection không ├── autoDisconnect // Auto-disconnect khi tất cả links lost │ // Multi-link handling: // - Vehicle có thể kết nối qua USB + Telemetry radio cùng lúc // - Nếu primary link mất → auto-switch sang secondary // - Commands gửi qua primary link // - Heartbeat được nhận từ any link

8. MAVLink Signing (Security)

MAVLink v2 hỗ trợ message signing để chống giả mạo.

// MAVLink Signing workflow: // 1. Both GCS and vehicle share a secret key (256-bit) // 2. Each message signed with SHA-256 truncated to 48-bit // 3. Timestamp prevents replay attacks // 4. Key exchange via SETUP_SIGNING command // Files: // - MAVLinkSigning.h → Signing state management // - MAVLinkSigningKeys.cc → Key storage // Signing is optional - unsigned messages still accepted // when signing not required (accept_unsigned_callback)

🔒 Security Note

MAVLink signing chỉ cung cấp authentication (xác thực nguồn gốc), KHÔNG cung cấp encryption (mã hóa nội dung). Payload message vẫn readable bởi bất kỳ ai intercept được.

9. MAVLink FTP

File transfer protocol qua MAVLink cho truy cập filesystem trên vehicle.

// MAVLink FTP operations: // - List directory contents on vehicle // - Download files (logs, parameters, ...) // - Upload files (waypoints, scripts, ...) // - Create/remove directories // - Burst transfer for large files // Used for: // 1. Log file download from vehicle SD card // 2. Parameter file upload/download // 3. Mission file transfer // 4. Firmware update (flashing)

10. Heartbeat System

// ═══════════ GCS → Vehicle ═══════════ // MultiVehicleManager phát HEARTBEAT mỗi 1 giây // trên TẤT CẢ active links // Heartbeat chứa: MAV_TYPE_GCS, MAV_AUTOPILOT_INVALID // ═══════════ Vehicle → GCS ═══════════ // Vehicle gửi HEARTBEAT mỗi 1 giây (configurable) // Chứa: vehicle type, autopilot type, base_mode, custom_mode // ═══════════ New Vehicle Detection ═══════════ // MAVLinkProtocol nhận HEARTBEAT từ unknown system ID // → emit vehicleHeartbeatInfo(sysid, compid, autopilot, type) // → MultiVehicleManager tạo Vehicle mới // → Vehicle begins parameter download // ═══════════ Connection Loss Detection ═══════════ // VehicleLinkManager kiểm tra heartbeat timeout: // - Timeout: 3.5 giây (3.5x heartbeat interval) // - No heartbeat → communicationLost = true // - UI hiện "Communication Lost" banner // - Tất cả links timeout → vehicle disconnected

// MAVLinkStreamConfig quản lý data stream rates // Gửi MAV_CMD_SET_MESSAGE_INTERVAL cho mỗi message type // Typical stream rates: // ATTITUDE → 10 Hz (mượt cho HUD) // GLOBAL_POSITION → 5 Hz (bản đồ tracking) // SYS_STATUS → 2 Hz (battery, mode) // GPS_RAW_INT → 3 Hz (GPS data) // RC_CHANNELS → 2 Hz (RC input) // SERVO_OUTPUT_RAW → 2 Hz (motor output) // ArduPilot has APMMavlinkStreamRateSettings // cho custom stream rate configuration

Communication Flow Summary

📋 Tóm tắt

  1. Physical Layer: Serial USB, WiFi UDP, TCP socket, Bluetooth
  2. Link Layer: LinkInterface implementations (SerialLink, UDPLink, ...)
  3. Protocol Layer: MAVLinkProtocol parse, validate, route
  4. Application Layer: Vehicle, ParameterManager, MissionManager xử lý messages
  5. Security: Optional MAVLink v2 signing
  6. Reliability: Heartbeat monitoring, auto-reconnect, multi-link redundancy
