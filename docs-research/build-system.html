2. CMake Configuration Architecture

2.1 Root CMakeLists.txt Flow

# Root CMakeLists.txt execution order: 1. cmake_minimum_required(3.25) 2. CMAKE_MODULE_PATH setup (cmake/, cmake/find-modules/, ...) 3. include(Helpers) # Utility functions 4. include(CustomOptions) # Default build options 5. Custom build check # Is custom/ directory present? 6. include(Git) # Extract version from git tags 7. Compiler caching (ccache) # Speed up rebuilds 8. project() declaration # Name, version, description 9. include(Toolchain) # C++20, compiler flags, LTO 10. find_package(Qt6) # Find Qt 6.10+ (20+ components) 11. qt_standard_project_setup # Qt6 project configuration 12. qt_add_executable() # Create main target 13. Platform config # Windows/macOS/Android/Linux 14. qt_add_qml_module() # QML module "QGC 1.0" 15. add_subdirectory(src) # Build source code 16. add_subdirectory(test) # Build tests (if enabled) 17. Translations # Internationalization 18. Install & packaging # Deployment

5. Custom Build System

Custom Builds (White-labeling)

QGC hỗ trợ custom builds cho third-party modifications. Chỉ cần tạo thư mục custom/.

# Custom build directory structure: custom/ ├── CMakeLists.txt # Custom build config ├── cmake/ │ └── CustomOverrides.cmake # Override build options ├── src/ │ ├── CustomPlugin.cc # Override QGCCorePlugin │ ├── CustomOptions.cc # Override QGCOptions │ └── ... ├── res/ # Custom resources (icons, images) └── android/ # Custom Android config # Root CMakeLists.txt auto-detects custom/ directory: if(IS_DIRECTORY "${CMAKE_SOURCE_DIR}/custom") set(QGC_CUSTOM_BUILD ON) include(CustomOverrides) endif()

💡 Custom Build Use Cases

  • Rebranding: Đổi tên, logo, màu sắc
  • Feature restriction: Ẩn/hiện tính năng
  • Custom firmware: Thêm support cho firmware riêng
  • Enterprise: Thêm authentication, fleet management

6. Platform-Specific Build

Windows

# cmake/platform/Windows.cmake # - WIN32 GUI application (no console window) # - OpenSSL bundled for HTTPS # - MSVC runtime configuration # - Windows icon (app.ico) và manifest cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_PREFIX_PATH=C:/Qt/6.10.3/msvc2022_64 cmake --build build --parallel

macOS

# cmake/platform/Apple.cmake # - MACOSX_BUNDLE → .app bundle # - Universal binary: x86_64h + arm64 # - Deployment target: configurable # - Code signing for notarization cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release -DQGC_MACOS_UNIVERSAL_BUILD=ON

Linux

# cmake/platform/Linux.cmake # - AppImage creation # - Wayland + X11 + EGL support # - Desktop file installation cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release -DQGC_BUILD_INSTALLER=ON

Android

# cmake/platform/Android.cmake # - Android SDK/NDK integration # - APK/AAB generation # - Min SDK: configurable (default 26) # - Target SDK: configurable (default 34) # - Gradle build integration cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release \ -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK/build/cmake/android.toolchain.cmake \ -DANDROID_ABI=arm64-v8a

8. Dependencies (CPM)

Package Management

QGC sử dụng CPM (CMake Package Manager) cho external dependencies.

# CPM source cache (avoid re-downloading) # Default: .cache/CPM/ in project root # Or set: $CPM_SOURCE_CACHE environment variable # Lock file: build/cpm-package-lock.cmake # Ensures reproducible builds # Key dependencies managed by CPM: # - MAVLink C library (message definitions) # - GStreamer (video streaming) # - Various Qt plugins
