1. Vehicle Module

Class Vehicle là trung tâm của QGC - đại diện cho một drone/vehicle đã kết nối hoặc offline.

Thành phần chính của Vehicle:

ComponentTypeMô tả
_parameterManagerParameterManager*Download/upload parameters từ vehicle
_vehicleLinkManagerVehicleLinkManager*Quản lý communication links
_firmwarePluginFirmwarePlugin*Firmware-specific behavior
_autopilotPluginAutoPilotPlugin*Vehicle setup UI
_missionManagerMissionManager*Mission upload/download
_geoFenceManagerGeoFenceManager*Geofence management
_rallyPointManagerRallyPointManager*Rally/safe points

FactGroups (Telemetry Data):

// Vehicle kế thừa từ FactGroup và chứa nhiều sub-groups Vehicle : public FactGroup ├── gps // GPS data (lat, lon, alt, hdop, vdop, ...) ├── gps2 // Secondary GPS ├── gpsAggregate // Combined GPS quality ├── wind // Wind speed/direction ├── vibration // Vibration levels (x, y, z) ├── temperature // Board/ambient temperature ├── clock // Vehicle clock ├── setpoint // Attitude setpoints ├── estimatorStatus // EKF status ├── terrain // Terrain height data ├── distanceSensors // Rangefinder data ├── localPosition // Local NED position ├── hygrometer // Humidity sensor ├── generator // Generator status ├── efi // Electronic Fuel Injection ├── batteries[] // Battery FactGroupListModel (dynamic) └── escs[] // ESC FactGroupListModel (dynamic)

Guided Mode Commands:

// Vehicle cung cấp high-level commands cho guided flight vehicle->guidedModeRTL(); // Return to Launch vehicle->guidedModeLand(); // Land at current position vehicle->guidedModeTakeoff(altitude); // Auto takeoff vehicle->guidedModeGotoLocation(coord); // Fly to GPS coordinate vehicle->guidedModeChangeAltitude(alt); // Change altitude vehicle->pauseVehicle(); // Pause/hold position vehicle->emergencyStop(); // Kill motors immediately vehicle->motorTest(motor, percent, secs); // Test individual motor

2. MultiVehicleManager

Singleton quản lý tất cả vehicles và xử lý vehicle selection.

MultiVehicleManager::instance() │ ├── activeVehicle // Vehicle đang được chọn (có thể null!) ├── vehicles // QmlObjectListModel chứa tất cả vehicles ├── offlineEditingVehicle // Vehicle ảo cho mission planning offline ├── selectedVehicles[] // Multi-select cho batch operations │ ├── parameterReadyVehicleAvailable // true khi có vehicle với params loaded │ ├── getVehicleById(id) // Lookup vehicle bằng MAVLink system ID ├── selectVehicle(v) // Add to selection ├── deselectVehicle(v) // Remove from selection └── setActiveVehicle(v) // Two-phase activation process

⚠️ Two-Phase Vehicle Activation

Khi setActiveVehicle() được gọi, QGC thực hiện 2 giai đoạn:

  1. Deactivate old vehicle: Lưu state, disconnect signals, clear UI
  2. Activate new vehicle: Load parameters, connect signals, update UI

Pattern này đảm bảo không bao giờ có 2 vehicles active cùng lúc.

3. FactSystem Module

Hệ thống quản lý toàn bộ parameters và settings. Bao gồm 4 class chính:

3.1 FactMetaData

// Metadata cho 1 parameter/setting - load từ JSON FactMetaData ├── type // valueTypeUint8, Float, Int32, String, Bool, ... ├── name // Tên parameter (e.g., "PARAM_NAME") ├── shortDescription // Mô tả ngắn ├── longDescription // Mô tả chi tiết ├── units // Đơn vị (m, m/s, deg, ...) ├── min / max // Giới hạn giá trị ├── defaultValue // Giá trị mặc định ├── decimalPlaces // Số chữ số thập phân ├── enumStrings[] // Label cho dropdown ├── enumValues[] // Value cho dropdown ├── bitmaskStrings[] // Label cho bitmask checkboxes ├── bitmaskValues[] // Value cho bitmask ├── rawTranslator // Function: raw → cooked ├── cookedTranslator // Function: cooked → raw ├── vehicleRebootRequired // Cần reboot vehicle sau khi thay đổi └── qgcRebootRequired // Cần restart QGC sau khi thay đổi

3.2 Fact

Fact ├── rawValue() // Giá trị gốc (int/float/string) ├── cookedValue() // Giá trị đã convert (với đơn vị display) ├── setRawValue(v) // Set giá trị → emit valueChanged ├── setCookedValue(v) // Set qua converter → emit valueChanged ├── enumOrValueString() // Trả về enum label hoặc value string ├── validate(value, coerce) // Validate giá trị ├── metaData() // Truy cập FactMetaData │ // Signals: ├── valueChanged(rawValue) // Khi value thay đổi ├── containerRawValueChanged(raw) // Cần gửi lên vehicle └── vehicleUpdated(rawValue) // ACK từ vehicle

3.3 ParameterManager

ParameterManager ├── parametersReady // true khi tất cả params đã download ├── missingParameters // true nếu có params không nhận được ├── loadProgress // 0.0 - 1.0 download progress ├── pendingWrites // Số params chờ gửi lên vehicle │ ├── getParameter(comp, name) // Lấy Fact* cho 1 parameter ├── parameterExists(comp, name) // Kiểm tra param có tồn tại ├── parameterNames(comp) // Danh sách tên params ├── refreshAllParameters() // Re-download tất cả ├── refreshParameter(comp, name) // Re-download 1 param └── resetAllToVehicleConfiguration() // Revert changes // Parameter caching: // - Params được cache theo vehicleId/componentId // - PX4 dùng hash-check để validate cache nhanh // - Nếu hash match → dùng cache, không cần re-download

💡 Parameter Workflow

  1. Vehicle connect → ParameterManager request tất cả params
  2. Vehicle gửi PARAM_VALUE messages → ParameterManager tạo Fact cho mỗi param
  3. Khi tất cả params nhận đủ → parametersReady = true
  4. User thay đổi param trong UI → Fact.containerRawValueChanged signal
  5. ParameterManager gửi PARAM_SET message lên vehicle
  6. Vehicle ACK → Fact.vehicleUpdated signal

4. FirmwarePlugin Module

Abstract layer cho firmware-specific behaviors. Mỗi firmware type (PX4, ArduPilot, Generic) có implementation riêng.

Key Virtual Methods:

MethodMô tả
flightModes()Danh sách flight modes có sẵn
flightMode(baseMode, customMode)Convert MAVLink mode → tên hiển thị
setFlightMode(modeName)Convert tên → MAVLink SET_MODE command
isCapable(capability)Kiểm tra firmware hỗ trợ feature
autopilotPlugin(vehicle)Factory method tạo setup UI plugin
pauseFlightMode()Tên flight mode pause (khác nhau giữa PX4/APM)
missionFlightMode()Tên flight mode cho mission execution
rtlFlightMode()Tên flight mode Return to Launch
remapParamName()Remap tên param theo firmware version

Implementations:

// PX4FirmwarePlugin // - Flight modes: Manual, Altitude, Position, Mission, Hold, Return, ... // - Supports: guided, takeoff, orbit, ROI, landing patterns // - Parameter caching with hash verification // ArduPilotFirmwarePlugin (APM) // - Sub-plugins per vehicle type: // ArduCopterFirmwarePlugin (Copter modes: Stabilize, AltHold, Loiter, ...) // ArduPlaneFirmwarePlugin (Plane modes: Manual, FBWA, Auto, ...) // ArduRoverFirmwarePlugin (Rover modes: Manual, Guided, Auto, ...) // ArduSubFirmwarePlugin (Sub modes: Manual, Stabilize, ...) // GenericFirmwarePlugin // - Minimal capabilities // - Used for unknown MAVLink vehicles

PERIPHERAL

7. Camera Module

Điều khiển camera onboard qua MAVLink Camera Protocol v2.

  • QGCCameraManager: Quản lý tất cả cameras trên vehicle
  • QGCCameraControl: Điều khiển 1 camera (photo, video, settings)
  • Camera settings: Exposure, white balance, resolution - tất cả là Facts
  • MAVLink Camera Protocol: CAMERA_INFORMATION, CAMERA_SETTINGS, IMAGE commands
PERIPHERAL

8. Gimbal Module

  • GimbalController: Quản lý gimbal orientation
  • Pitch/Yaw control qua MAVLink GIMBAL_MANAGER messages
  • ROI (Region of Interest) tracking
  • Gimbal device protocol support
PERIPHERAL

9. GPS / RTK Module

  • GPSManager: Quản lý GPS base station cho RTK
  • NTRIP: NTRIP client cho RTK correction data từ internet
  • RTK: Real-Time Kinematic positioning với centimeter-level accuracy
  • Support u-blox GPS hardware trực tiếp qua USB
PERIPHERAL

10. VideoManager Module

  • VideoManager: Quản lý video streaming từ vehicle
  • GStreamer: Backend video decoding (RTSP, UDP, TCP)
  • VideoReceiver: Nhận và decode video stream
  • Support recording video locally
  • Support UVC cameras (USB webcam trực tiếp)
  • Thermal/IR camera overlay support
INPUT

11. Joystick Module

  • JoystickManager: Detect và quản lý connected joysticks
  • Joystick: SDL-based joystick input
  • Axis mapping: Roll, Pitch, Yaw, Throttle
  • Button mapping: Flight modes, camera trigger, gimbal
  • Deadzone và exponential curve configuration
  • Calibration wizard
