1. Singleton Pattern

QGC sử dụng Singleton rộng rãi cho các manager classes. Các singleton được truy cập qua ::instance() static method.

Singleton ClassTrách nhiệm
MultiVehicleManagerQuản lý tất cả connected vehicles
LinkManagerQuản lý communication links
MAVLinkProtocolParse & route MAVLink messages
SettingsManagerQuản lý 25+ settings groups
QGroundControlQmlGlobalQML global context
// Cách truy cập singleton Vehicle* vehicle = MultiVehicleManager::instance()->activeVehicle(); LinkManager::instance()->createConnectedLink(config); SettingsManager::instance()->appSettings()->offlineEditingFirmwareClass();

💡 QML Singleton

Một số class sử dụng QML_SINGLETON macro để expose singleton trực tiếp cho QML mà không cần qua context property.

Behavioral / Architectural

2. Fact System Pattern (Unique to QGC)

Đây là design pattern quan trọng nhất và độc đáo nhất của QGC. Fact System là một hệ thống metadata-driven cho toàn bộ vehicle parameters và app settings.

Kiến trúc Fact System:

// Hierarchy: FactMetaData → Fact → FactGroup → Vehicle ┌─────────────────────────────────┐ │ FactMetaData │ ← JSON metadata (type, min, max, units, enums) │ - type, min, max, defaultValue │ │ - shortDescription, units │ │ - enumStrings, enumValues │ │ - bitmaskStrings, bitmaskValues│ └──────────┬──────────────────────┘ │ attached to ▼ ┌─────────────────────────────────┐ │ Fact │ ← Single value container │ - rawValue / cookedValue │ ← Dual value: raw (storage) vs cooked (display) │ - validate() │ │ - enumOrValueString() │ │ - deferredValueChangeSignal() │ ← Rate-limited UI updates └──────────┬──────────────────────┘ │ contained in ▼ ┌─────────────────────────────────┐ │ FactGroup │ ← Hierarchical container │ - facts: Map<name, Fact*> │ │ - groups: Map<name, FactGroup*>│ │ - handleMessage(mavlink_msg) │ ← Direct MAVLink processing! │ - setLiveUpdates(bool) │ └──────────┬──────────────────────┘ │ attached to ▼ ┌─────────────────────────────────┐ │ Vehicle │ │ - gps FactGroup │ │ - battery FactGroup │ │ - wind FactGroup │ │ - temperature FactGroup │ │ - ... 15+ FactGroups │ └─────────────────────────────────┘

Dual Value Concept (rawValue vs cookedValue):

🔧 rawValue

Giá trị gốc từ vehicle/MAVLink. Dùng cho storage và protocol.

// Altitude in meters (raw) fact->rawValue().toDouble(); // 100.0

👁️ cookedValue

Giá trị đã convert cho hiển thị (có đơn vị). Dùng cho UI.

// Altitude in feet (cooked) fact->cookedValue().toDouble(); // 328.08

Metadata JSON:

// *.FactMetaData.json - Mỗi parameter được mô tả bằng JSON { "name": "PARAM_NAME", "type": "Float", "shortDescription": "Parameter description", "min": 0, "max": 100, "defaultValue": 50, "units": "m/s", "decimalPlaces": 2 }
Structural

3. Plugin / Strategy Pattern

QGC sử dụng FirmwarePlugin pattern để abstract firmware-specific behavior. Đây là sự kết hợp giữa Strategy và Abstract Factory pattern.

// Plugin hierarchy ┌───────────────────┐ │ FirmwarePlugin │ ← Abstract base: flight modes, capabilities, params │ (virtual methods) │ └─────────┬─────────┘ │ ┌─────┼────────┐ ▼ ▼ ▼ ┌───────┐┌───────┐┌─────────┐ │PX4 ││APM ││Generic │ │Plugin ││Plugin ││Plugin │ └───┬───┘└───┬───┘└─────────┘ │ │ ▼ ▼ ┌───────┐┌───────────────────────┐ │PX4 ││APMFlightModes │ │Flight ││APMSensors │ │Modes ││APMSafety │ └───────┘│APMPower │ │APMSubFrameComponent │ └───────────────────────┘

FirmwarePlugin Capabilities:

enum class FirmwareCapabilities { SetFlightModeCapability, // Can change flight mode PauseVehicleCapability, // Can pause in-flight GuidedModeCapability, // Supports guided navigation TakeoffVehicleCapability, // Supports auto takeoff OrbitModeCapability, // Can orbit a point ROIModeCapability, // Region of Interest tracking ChangeHeadingCapability, // Can change heading in-flight };

Parameter Remapping (Version-aware):

// FirmwarePlugin handles parameter renaming across firmware versions // Example: ArduPilot renamed TUNE_LOW → TUNE_MIN in v4.0 virtual QString remapParamNameHigestAvailableVersion( int majorVersion, int minorVersion, const QString& paramName);
Behavioral

4. Observer Pattern (Qt Signal/Slot)

QGC tận dụng Qt Signal/Slot mechanism - một implementation mạnh mẽ của Observer pattern. Đặc biệt với Q_PROPERTY NOTIFY cho automatic QML binding.

// C++ Class class Vehicle : public FactGroup { Q_OBJECT // Q_PROPERTY tự động tạo observer pattern cho QML Q_PROPERTY(bool armed READ armed NOTIFY armedChanged) Q_PROPERTY(QString flightMode READ flightMode NOTIFY flightModeChanged) Q_PROPERTY(QGeoCoordinate coordinate READ coordinate NOTIFY coordinateChanged) signals: void armedChanged(bool armed); void flightModeChanged(const QString& flightMode); void coordinateChanged(QGeoCoordinate coordinate); }; // QML tự động subscribe - không cần manual connect // QGCLabel { text: vehicle.flightMode } // Auto-updates!

Deferred Signal Pattern:

// Fact sử dụng deferred signals để tránh flood UI khi telemetry update nhanh // Thay vì emit mỗi lần value thay đổi, nó batch signals fact->setDeferredValueChangeSignal(true); // ... nhiều updates ... fact->clearDeferredValueChangeSignal(); // Emit 1 lần duy nhất
Structural

Creational

6. Abstract Factory (AutoPilotPlugin)

Mỗi firmware type tạo ra bộ UI components riêng thông qua AutoPilotPlugin factory:

// FirmwarePlugin đóng vai trò Abstract Factory AutoPilotPlugin* plugin = vehicle->firmwarePlugin()->autopilotPlugin(vehicle); // Mỗi firmware trả về các VehicleComponents khác nhau: // PX4: // - AirframeComponent // - SensorsComponent // - RadioComponent // - FlightModesComponent // - SafetyComponent // - PowerComponent // ArduPilot: // - APMFlightModesComponent // - APMSensorsComponent // - APMRadioComponent // - APMSafetyComponent // - APMPowerComponent // - APMMotorComponent // - APMSubFrameComponent (submarine only)
Memory Management

7. Shared Pointer Pattern (Links)

Communication links sử dụng std::shared_ptr cho lifetime management:

// LinkInterface sử dụng shared_ptr để tránh use-after-free using SharedLinkInterfacePtr = std::shared_ptr<LinkInterface>; // Vehicle giữ shared_ptr đến link // Khi vehicle bị xóa, link ref count giảm // Link chỉ bị destroy khi không còn ai reference // Multi-link support: 1 vehicle có thể dùng nhiều links // VehicleLinkManager quản lý primary/secondary links
Behavioral

8. Command Pattern (Mission Commands)

Mission commands được tổ chức theo Command pattern với metadata-driven UI:

// MissionCommandTree: Registry of all MAVLink commands // MissionCommandUIInfo: Metadata for command UI rendering // MissionCommandList: Available commands per vehicle type // JSON defines command UI: { "id": 16, // MAV_CMD_NAV_WAYPOINT "rawName": "MAV_CMD_NAV_WAYPOINT", "friendlyName": "Waypoint", "description": "Travel to a position", "param1": { "label": "Hold", "units": "secs", "default": 0, "decimalPlaces": 0 } }
Communication

9. Heartbeat / Watchdog Pattern

QGC sử dụng heartbeat mechanism cho vehicle connection monitoring:

// VehicleLinkManager: heartbeat monitoring // - GCS gửi heartbeat mỗi 1 giây // - Vehicle phải respond trong 3.5 giây // - Quá timeout → communicationLost = true // - Tất cả links lost → vehicle disconnected // MultiVehicleManager phát heartbeat: // init() → start heartbeat timer // Timer fires → send HEARTBEAT on all active links // New vehicle detection: // MAVLinkProtocol nhận HEARTBEAT từ unknown sysid // → emit vehicleHeartbeatInfo() // → MultiVehicleManager tạo Vehicle mới
Architectural

10. Model-View Pattern (QML Integration)

QGC bridge C++ models sang QML views thông qua QmlObjectListModel:

// C++ side: QmlObjectListModel kế thừa QAbstractListModel // Cung cấp list data cho QML ListView/Repeater class QmlObjectListModel : public QAbstractListModel { // Tự động notify QML khi items thêm/xóa/thay đổi // QML Repeater tự động update khi model thay đổi }; // QML side: // Repeater { // model: QGroundControl.multiVehicleManager.vehicles // delegate: VehicleIndicator { vehicle: object } // }
