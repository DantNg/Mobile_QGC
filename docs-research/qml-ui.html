1. QML/C++ Integration Architecture

QGC sử dụng QML cho toàn bộ UI, với C++ backend expose qua Qt's meta-object system.

1.1 C++ → QML Exposure

// 3 cách expose C++ class cho QML: // 1. QML_ELEMENT - Creatable trong QML class MyWidget : public QQuickItem { Q_OBJECT QML_ELEMENT // QML: MyWidget { } }; // 2. QML_SINGLETON - Singleton access class QGroundControl : public QObject { Q_OBJECT QML_SINGLETON // QML: QGroundControl.multiVehicleManager }; // 3. QML_UNCREATABLE - Chỉ từ C++, không tạo trong QML class Vehicle : public FactGroup { Q_OBJECT QML_UNCREATABLE("") // Chỉ truy cập qua property };

Q_PROPERTY System:

// Q_PROPERTY = Reactive data binding C++ → QML // Khi NOTIFY signal phát → QML tự động re-render class Vehicle { // Read-Write property Q_PROPERTY(bool armed READ armed WRITE setArmed NOTIFY armedChanged) // Read-Only property Q_PROPERTY(QString flightMode READ flightMode NOTIFY flightModeChanged) // Constant property (never changes) Q_PROPERTY(ParameterManager* parameterManager READ parameterManager CONSTANT) // Method callable từ QML Q_INVOKABLE void guidedModeTakeoff(double altitude); }; // QML usage - TỰ ĐỘNG reactive: // QGCLabel { text: vehicle.flightMode } // Auto-updates khi mode thay đổi! // QGCButton { onClicked: vehicle.guidedModeTakeoff(10) }

// QGCPalette.cc - Central color management // Hỗ trợ Light và Dark themes // TUYỆT ĐỐI KHÔNG hardcode colors trong QML! // QML usage: import QGroundControl.Palette Item { QGCPalette { id: qgcPal } Rectangle { color: qgcPal.window // Background color } QGCLabel { color: qgcPal.text // Text color } QGCButton { // Button tự động sử dụng palette // Không cần set color manually } } // Available palette colors: // qgcPal.window - Window background // qgcPal.windowShade - Slightly different shade // qgcPal.windowShadeDark - Darker shade // qgcPal.text - Normal text // qgcPal.warningText - Warning text // qgcPal.button - Button background // qgcPal.buttonHighlight - Button hover // qgcPal.buttonText - Button text // qgcPal.primaryButton - Primary action button // qgcPal.mapButton - Map overlay button // qgcPal.alertBackground - Alert/error background // ... many more

4.2 ScreenTools (Responsive Sizing)

// ScreenTools.qml - DPI-aware sizing // TUYỆT ĐỐI KHÔNG hardcode pixel values! // QML usage: import QGroundControl.ScreenTools Item { // Font-relative sizing: width: ScreenTools.defaultFontPixelWidth * 30 // ~30 characters wide height: ScreenTools.defaultFontPixelHeight * 2 // ~2 lines tall // Standard margins: margins: ScreenTools.defaultFontPixelWidth // Font sizes: QGCLabel { font.pointSize: ScreenTools.defaultFontPointSize // Normal } QGCLabel { font.pointSize: ScreenTools.largeFontPointSize // Large } QGCLabel { font.pointSize: ScreenTools.smallFontPointSize // Small } // Mobile detection: visible: ScreenTools.isMobile ? false : true } // Key ScreenTools properties: // defaultFontPixelWidth - Width of one character // defaultFontPixelHeight - Height of one line // defaultFontPointSize - Base font size // largeFontPointSize - Large font // smallFontPointSize - Small font // isMobile - Mobile platform detection // isAndroid, isiOS - Platform checks

5. QML File Structure Convention

Standard QML File Layout

import QtQuick import QtQuick.Controls import QtQuick.Layouts import QGroundControl import QGroundControl.Controls import QGroundControl.ScreenTools import QGroundControl.Palette Item { id: root // 1. Kích thước & anchors width: ScreenTools.defaultFontPixelHeight * 10 height: ScreenTools.defaultFontPixelHeight * 5 // 2. Public properties property var vehicle: QGroundControl.multiVehicleManager.activeVehicle property int myProperty: 0 // 3. Private properties (underscore prefix) readonly property bool _isValid: vehicle !== null readonly property real _margins: ScreenTools.defaultFontPixelWidth // 4. Palette QGCPalette { id: qgcPal } // 5. Signals signal clicked() // 6. Functions function doSomething() { // logic } // 7. Visual children ColumnLayout { anchors.fill: parent spacing: _margins QGCLabel { text: _isValid ? vehicle.flightMode : qsTr("No Vehicle") } QGCButton { text: qsTr("Click Me") // qsTr() for i18n! onClicked: root.clicked() } } // 8. Connections (Qt6 function syntax) Connections { target: vehicle function onArmedChanged() { console.log("Armed state changed") } } // 9. Component.onCompleted Component.onCompleted: { // initialization } }

8. Golden Rules Cho QML Development

⚠️ KHÔNG BAO GIỜ

  • Hardcode sizes: Dùng ScreenTools.defaultFontPixelHeight/Width
  • Hardcode colors: Dùng QGCPalette
  • Dùng Qt controls trực tiếp: Dùng QGC controls (QGCButton, QGCLabel, ...)
  • Bỏ qsTr(): Tất cả user-visible strings phải wrap bằng qsTr()
  • Quên null-check vehicle: Luôn kiểm tra vehicle !== null
  • Dùng Connections syntax cũ: Dùng function onSignal() {} thay vì onSignal: {}

✅ LUÔN LUÔN

  • Dùng ScreenTools cho mọi sizing
  • Dùng QGCPalette cho mọi colors
  • Dùng QGC custom controls thay vì Qt controls
  • Wrap strings với qsTr() cho i18n
  • Null-check activeVehicle trước khi sử dụng
  • Dùng Qt6 Connections syntax: function onSignalName() {}
  • Đẩy logic phức tạp về C++ (Q_INVOKABLE methods)
  • Follow file structure convention (imports → id → sizes → properties → children)
