3. Application Lifecycle

Flow khởi động ứng dụng:

main.cc
  │
  ▼
QGCApplication (kế thừa QApplication)
  │
  ├── init()
  │    ├── Settings version check
  │    ├── SettingsManager khởi tạo
  │    ├── LinkManager khởi tạo
  │    └── MultiVehicleManager khởi tạo
  │
  ├── _initForNormalAppBoot()
  │    ├── QQmlApplicationEngine tạo
  │    ├── QML context properties đăng ký
  │    ├── Singleton managers expose to QML
  │    ├── Video/GStreamer init
  │    └── Root QML window load
  │
  └── exec() → Qt Event Loop
       ├── MAVLink message processing
       ├── Vehicle heartbeat monitoring
       ├── Link connection management
       └── QML UI rendering

4. Data Flow Chính

4.1 MAVLink Message Flow (Nhận dữ liệu từ drone)

Drone/Vehicle (Hardware)
      │
      │ MAVLink bytes (Serial/UDP/TCP)
      ▼
┌──────────────┐
│ LinkInterface │ ← SerialLink / UDPLink / TCPLink
│ (bytesReceived│
│    signal)    │
└──────┬───────┘
       ▼
┌──────────────────┐
│ MAVLinkProtocol   │ ← Singleton: parse, validate, track sequence
│ (messageReceived  │
│    signal)        │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ Vehicle           │ ← Route message to right vehicle by sysid
│ _mavlinkMessage   │
│ Handler()         │
└──────┬───────────┘
       │
       ├──► ParameterManager  → Fact.rawValueChanged()
       ├──► MissionManager    → Mission progress update
       ├──► FactGroups        → Telemetry (GPS, battery, ...)
       ├──► VehicleLinkManager→ Heartbeat tracking
       └──► StatusTextHandler → Status messages
              │
              ▼
         QML UI Update (via Q_PROPERTY NOTIFY signals)

4.2 Command Flow (Gửi lệnh đến drone)

QML UI (Button click / Slider change)
      │
      │ Q_INVOKABLE / Signal
      ▼
┌──────────────────┐
│ Vehicle           │ ← guidedModeTakeoff(), setFlightMode(), etc.
│                   │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ FirmwarePlugin    │ ← Encode to correct MAVLink format per firmware
│                   │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│ LinkInterface     │ ← _writeBytes() → Serial/UDP/TCP
│                   │
└──────────────────┘
       │
       ▼
  Drone/Vehicle

6. Multi-Vehicle Architecture

💡 Nguyên tắc quan trọng nhất

QGC hỗ trợ kết nối nhiều vehicle cùng lúc. Mọi code đều phải null-check activeVehicle() và không được giả định chỉ có 1 vehicle.

┌─────────────────────────────┐
│    MultiVehicleManager       │ ← Singleton, quản lý tất cả vehicles
│    (activeVehicle property)  │
├─────────────────────────────┤
│                              │
│  vehicles: QmlObjectListModel│
│  ┌──────────┐ ┌──────────┐  │
│  │ Vehicle 1 │ │ Vehicle 2 │  │
│  │ (MAV ID 1)│ │ (MAV ID 2)│  │
│  └───┬──────┘ └───┬──────┘  │
│      │             │          │
│  ┌───┴──────┐ ┌───┴──────┐  │
│  │Link A     │ │Link B     │  │
│  │(Serial)   │ │(UDP)      │  │
│  └──────────┘ └──────────┘  │
│                              │
│  offlineEditingVehicle       │ ← For mission planning without connection
│  selectedVehicles[]          │ ← Multi-select support
└─────────────────────────────┘

7. Offline vs Connected Mode

Vehicle class có 2 constructors hoàn toàn khác nhau:

Aspect Connected Vehicle Offline Vehicle
Constructor Vehicle(LinkInterface*, vehicleId, autopilot, type) Vehicle(autopilot, type)
Mục đích Điều khiển drone thật Lập kế hoạch mission offline
Parameters Download từ vehicle Default values từ metadata
Link Serial/UDP/TCP Không có
Telemetry Real-time updates Không có

⚠️ Lưu ý quan trọng

Mọi code truy cập vehicle đều phải kiểm tra null trước khi sử dụng:

Vehicle* vehicle = MultiVehicleManager::instance()->activeVehicle(); if (!vehicle) return; // Safe to use vehicle here
